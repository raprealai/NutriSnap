### The Standard Project Flow
Every project follows this path:

```
1. GOVERNANCE (00)
   → New project arrives → Run approval check
   → Output: Project Approval Brief

2. COE (01)
   → Receives approved project → Builds Project Plan
   → Assigns pods → Sets quality standard

3. LEGAL COMPLIANCE (02/legal_compliance)
   → Runs risk assessment immediately
   → Output: Compliance clearance or flag

4. AI PRODUCT MANAGER (02/ai_product_manager)
   → Writes full PRD
   → Shares user stories with UX Designer

5. UX DESIGNER (02/ux_designer)
   → Designs conversation flow / interface
   → Writes bot personality (if applicable)

6. DATA ENGINEER (02/data_engineer)   [if data is involved]
   → Cleans and structures data
   → Hands off to ML Engineer

7. ML ENGINEER (02/ml_engineer)
   → Builds the AI system / prompt / integration
   → Hands off to MLOps for deployment

8. DATA SCIENTIST (02/data_scientist)
   → Evaluates the build (10+ test cases)
   → Approves or flags issues

9. MLOPS ENGINEER (02/mlops_engineer)
   → Deploys the system
   → Sets up monitoring and automation

10. COE (01) — Quality Review
    → Final check before client delivery

11. GOVERNANCE (00) — Sign-off
    → Approves delivery

12. BUSINESS INTEGRATION (05/business_integration)
    → Calculates ROI post-delivery
    → Plans recurring revenue opportunities
```
